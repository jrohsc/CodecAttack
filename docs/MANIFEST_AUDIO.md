# Audio data manifest

This repository contains all data needed to verify the paper's results that are safe to redistribute. Two categories of audio carriers are intentionally **not** included; this file documents what they are and how to obtain them locally.

## What is included

| Folder | Source | Count | Size | Notes |
|---|---|---|---|---|
| `data/speech/01_finance/audio/` | Microsoft Edge TTS | 50 | 7.8 MB | Banking voice carriers, 5 scripts × 10 voices |
| `data/speech/02_interview/en/audio/` | Microsoft Edge TTS | 25 | 4.2 MB | English interview carriers |
| `data/speech/02_interview/zh/audio/` | Microsoft Edge TTS | 18 | ~3 MB | Mandarin interview carriers (`chinese_bad_*`, `chinese_murmur_*`, `chinese_v2_*`); all TTS-generated |
| `data/speech/04_podcast_advert/audio/` | Microsoft Edge TTS | 4 | 3.0 MB | Podcast / advertisement carriers |
| `data/music/ai_generated/` | Suno (text-to-music) | 8 | 28 MB | Generated by us via Suno; used as S3a (AI-detection bypass) carriers |
| `data/music/03a_ai_detection_bypass/carrier_to_target.json` | this repo | 1 | <1 KB | S3a target-text mapping |
| `data/music/03b_copyright_bypass/carrier_to_target.json` | this repo | 1 | <1 KB | S3b target-text mapping (audio not redistributed; see below) |
| `data/speech/<scenario>/scripts/` | this repo | varies | tiny | Plain-text source scripts that the TTS was generated from |
| `data/speech/<scenario>/carrier_to_target.json` | this repo | one per scenario | <100 KB total | Per-pair target text and category metadata |

All speech audio above was generated locally with Microsoft Edge TTS (`edge-tts` CLI). All music carriers in `ai_generated/` were generated by us via the Suno text-to-music interface. Either is safe to redistribute under typical research-use terms.

### 2. MagicData Mandarin clips (S2 interview)

`data/speech/02_interview/carrier_to_target.json` lists six carriers named `chinese_magicdata_*.wav`. These are real human-speech recordings from the MagicData Mandarin Conversational Speech corpus (commercial dataset from Magic Data Technology Co., Ltd.) and are not included here. To reproduce the S2 Mandarin condition with those six carriers, license the corpus from Magic Data, extract the indicated clip indices (`m_q1_idx0`, `m_q2_idx1`, `f_q3_idx2`, `m_q1_idx3`, `f_q2_idx4`, `f_q3_idx5`), and place them at `data/speech/02_interview/zh/audio/`. The remaining 18 Mandarin carriers in this folder (`chinese_bad_*`, `chinese_murmur_*`, `chinese_v2_*`) are all TTS-generated and included.

### 3. Adversarial waveforms

For all scenarios, we ship the JSON / JSONL eval rollups (per-pair WER, per-channel ASR, per-cell PESQ / LSD / SNR / dLUFS) under `data/eval_rollups/`. We do not ship the ~10k+ attacked WAVs themselves; they regenerate deterministically from the included scripts.
